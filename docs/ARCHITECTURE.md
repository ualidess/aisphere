# AI Sphere Avatar — схема взаимодействия контейнеров и внешнего мира

## Принцип

- **Наружу торчит только один порт** — Nginx (reverse proxy). Это единственная точка входа.
- **Backend, БД и уборщик не публикуют порты наружу** — общаются только внутри приватной Docker-сети.
- **К внешним API (OpenAI, ElevenLabs) ходит только backend**, по исходящим HTTPS-запросам. Браузер напрямую к ним не обращается (ключи не утекают на фронт).
- Браузер общается с backend **только через Nginx** (`/api/*`), поэтому фронтенду не нужно знать адрес backend.

## Кто с кем общается

| Источник            | Назначение            | Протокол / порт         | Публикуется наружу? |
|---------------------|-----------------------|-------------------------|---------------------|
| Браузер             | nginx                 | HTTPS :443 (HTTP :80 → редирект на 443) | **Да** (единственный вход) |
| nginx               | backend               | HTTP :8000 (внутр. сеть) | Нет                |
| backend             | postgres              | TCP :5432 (внутр. сеть)  | Нет                |
| cleaner             | postgres              | TCP :5432 (внутр. сеть)  | Нет                |
| backend             | OpenAI / ElevenLabs   | исходящий HTTPS :443     | — (исходящее)      |
| postgres            | pgdata (volume)       | файловая система         | Нет                |

> «Уборщик» (`cleaner`) можно сделать не отдельным контейнером, а фоновой задачей внутри `backend`. Тогда контейнеров три: `nginx`, `backend`, `postgres`. Отдельный контейнер удобнее, если захотим запускать уборку независимо и масштабировать backend.

## Правила доступа (важно для Фазы 7 — безопасность)

1. **Только nginx публикует порт** наружу (`ports:` в docker-compose есть только у nginx). У backend и postgres портов наружу нет — они доступны лишь по имени сервиса внутри сети Docker.
2. **TLS заканчивается на nginx** (HTTPS снаружи). Внутри сети — обычный HTTP, т.к. сеть приватная.
3. **БД никогда не смотрит в интернет.** Доступ к ней — только из backend и cleaner.
4. **Ключи внешних API хранятся только в backend** (переменные окружения), браузер их не видит.
5. **CORS** на backend ограничен доменом фронтенда; rate limiting — на nginx и/или backend.
6. Браузер ↔ backend идёт строго через `/api/*` на nginx — фронт не знает внутренних адресов.
