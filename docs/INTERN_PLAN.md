# AI Sphere Avatar — план доработки до MVP (для стажёров)

## Цель

Превратить текущий прототип (голосовой вопрос-ответ через 3D-сферу) в **минимальный рабочий продукт**:

- голосовое общение с **ChatGPT** через сферу;
- **STT и TTS через ElevenLabs** (Whisper и OpenAI TTS убираем);
- **пользователи** со своей историей;
- **несколько чатов на одного пользователя** с возможностью переключения;
- **память диалога** (бот помнит контекст в рамках чата);
- **асинхронный** бэкенд;
- **логирование** и хранение истории в **БД**;
- **защита от нагрузок и взлома** (аутентификация, rate limiting, валидация).

Внешний API `policy_router_api` и связанную с ним external-сеть **полностью убираем**.

---

## Целевая архитектура (общая картина)

```
Браузер (Three.js сфера + экран входа + список чатов)
   │  HTTPS, JWT в заголовке
   ▼
Async backend (FastAPI)
   ├── Auth        (регистрация / вход / JWT)
   ├── Chats       (создать / список / переключить / переименовать / удалить)
   ├── Messages    (история сообщений чата)
   ├── /stt   → ElevenLabs Speech-to-Text (Scribe)
   ├── /chat  → OpenAI Chat Completions (с историей чата)
   └── /tts   → ElevenLabs Text-to-Speech
   │
   ▼
PostgreSQL (users, chats, messages)
```

### Рекомендуемый стек (решено за стажёров — не менять без согласования)

- **Backend:** FastAPI + Uvicorn (async-нативный, вместо синхронного Flask).
- **БД:** PostgreSQL. Доступ — async (SQLAlchemy 2.0 async + asyncpg). Миграции — Alembic.
- **Аутентификация:** JWT (access-токен), пароли — хэш (bcrypt/argon2).
- **HTTP к внешним API:** async-клиент (httpx).
- **Rate limiting:** slowapi (или аналог).
- **Логи:** структурированные (JSON), с request-id.
- **Внешние сервисы:** OpenAI (чат), ElevenLabs (STT + TTS).

## Фаза 4 — Чаты, история и память диалога

> Цель: несколько чатов на пользователя + бот помнит контекст.

- [ ] CRUD чатов: `POST /chats` (создать), `GET /chats` (список), `GET /chats/{id}` (сообщения), `PATCH /chats/{id}` (переименовать), `DELETE /chats/{id}`.
- [ ] При запросе к `/chat` сообщение привязывается к конкретному `chat_id`.
- [ ] Перед вызовом ChatGPT подгружать историю сообщений чата и передавать её в запрос (память диалога).
- [ ] Сохранять в БД и вопрос пользователя, и ответ ассистента.
- [ ] Ограничить длину передаваемой истории (последние N сообщений или по токенам), чтобы не раздувать запрос.
- [ ] Авто-название чата по первому сообщению (опционально, по согласованию).
- [ ] **Хранение 3 суток (72 ч):** чаты и сообщения старше 72 ч от последней активности удаляются автоматически.
  - Фоновая задача-уборщик (периодический запуск, напр. раз в час) удаляет `chats` с `updated_at` старше 72 ч, каскадно с их `messages`.
  - Срок считать от последней активности (`updated_at` обновляется при каждом новом сообщении), а не от создания чата.
  - На уровне БД: индекс по `updated_at` и каскадное удаление сообщений (`ON DELETE CASCADE`).
  - Фронтенд: показать пользователю, что чаты живут 72 часа.

**Критерий приёмки:** у пользователя несколько чатов, переключение сохраняет контекст; бот учитывает предыдущие реплики внутри чата; чаты старше 72 ч автоматически исчезают вместе с сообщениями.

## Фаза 7 — Защита от нагрузок и взлома

> Цель: «нормальный минимальный продукт», а не открытый прокси к платным API.

- [ ] **Rate limiting** на тяжёлые/платные эндпоинты (`/stt`, `/chat`, `/tts`, `/auth/login`) — по пользователю и по IP.
- [ ] **Валидация входных данных:** лимит размера загружаемого аудио, максимальная длина текста для TTS, проверка типов/форматов (Pydantic-схемы).
- [ ] **Ограничение размера запроса** на уровне сервера и nginx.
- [ ] **CORS**: разрешить только нужные домены (сейчас открыт для всех).
- [ ] **Секреты только в переменных окружения**, ничего в коде/репозитории.
- [ ] **Заголовки безопасности** и **HTTPS** в nginx (security headers; TLS для прод/внешнего доступа — микрофон требует защищённый контекст).
- [ ] Брутфорс-защита на логине (rate limit + задержки), безопасные сообщения об ошибках (не раскрывать, существует ли пользователь).
- [ ] (Опционально) лимит расходов на пользователя/в сутки на платные API.

**Критерий приёмки:** превышение лимитов даёт 429; некорректные/слишком большие запросы отклоняются; эндпоинты закрыты авторизацией; CORS ограничен.
